IBKR as Primary Engine — Not a Fallback
Eliminate intraday connectivity pain by restricting IBKR to two narrow, scheduled windows per day.
💡
Core insight: IBKR's TWS API "pain" is entirely an intraday problem — persistent connections, pacing limits, reconnect logic, snapshot conflicts. None of those constraints apply when you run two 5–10 minute batch jobs: one at 9:31 AM and one at 4:05 PM. The API becomes simple, predictable, and free.

End-of-Day Batch (10 min) — The IVR Engine

Two parallel calls per ETF: (1) composite OPTION_IMPLIED_VOLATILITY historical bar — one clean number for the IVR database; (2) ATM chain snapshot for Greeks archival.

Python script reads today's IV from SQLite → ranks vs. prior 252 days → writes IV Rank + IV Percentile to scanner DB. Gate check fires: if IVR > 30% → mark ETF as eligible for premium selling. Dashboard updates automatically.

EOD Collector — Python Script
Runs at 4:05 PM ET. Collects composite IV + ATM Greeks. Computes IVR/IVP. Disconnects cleanly.
Python — ibkr_eod_collector.py 
# ibkr_eod_collector.py  — Run via cron: 5 16 * * 1-5
# Requires: ib_insync, SQLite3, IB Gateway running on port 7497

from ib_insync import IB, Stock, Option, util
import sqlite3, datetime, numpy as np, time

ETF_UNIVERSE = [
    'XLK','XLY','XLP','XLV','XLF','XLI','XLE','XLU',
    'XLB','XLRE','XLC','MDY','IWM','SCHB','QQQ','TQQQ'
]

def run_eod_collection():
    ib = IB()
    ib.connect('127.0.0.1', 7497, clientId=10)   # Gateway port
    conn = sqlite3.connect('options_data.db')

    today = datetime.date.today()

    for ticker in ETF_UNIVERSE:
        contract = Stock(ticker, 'SMART', 'USD')
        ib.qualifyContracts(contract)

        # ── Step 1: Composite OPTION_IMPLIED_VOLATILITY (the IVR engine number)
        bars = ib.reqHistoricalData(
            contract,
            endDateTime='',                  # now
            durationStr='1 D',
            barSizeSetting='1 day',
            whatToShow='OPTION_IMPLIED_VOLATILITY',
            useRTH=1,
            formatDate=1
        )
        if bars:
            iv_today = bars[-1].close     # composite IV (e.g., 0.22 = 22% annualised)
            conn.execute(
                "INSERT OR REPLACE INTO iv_history VALUES (?,?,?)",
                (ticker, str(today), iv_today)
            )

        # ── Step 2: Compute IVR and IVP from 252-day rolling window
        rows = conn.execute(
            "SELECT iv FROM iv_history WHERE ticker=? ORDER BY date DESC LIMIT 252",
            (ticker,)
        ).fetchall()
        iv_series = [r[0] for r in rows]

        if len(iv_series) >= 30:   # min 30 days to compute meaningful rank
            iv_rank = (iv_today - min(iv_series)) / (max(iv_series) - min(iv_series)) * 100
            iv_pct  = round(sum(iv < iv_today for iv in iv_series) / len(iv_series) * 100, 1)
            conn.execute(
                "INSERT OR REPLACE INTO ivr_daily VALUES (?,?,?,?,?)",
                (ticker, str(today), iv_today, iv_rank, iv_pct)
            )
            print(f"{ticker}: IV={iv_today:.3f} | IVR={iv_rank:.1f}% | IVP={iv_pct:.1f}%")

        time.sleep(0.5)   # gentle pacing — well under IBKR's 50req/10s limit

    conn.commit()
    conn.close()
    ib.disconnect()
    print("EOD collection complete. IVR database updated.")


# ── One-time backfill: run once to seed the full 252-day history
def backfill_iv_history():
    ib = IB()
    ib.connect('127.0.0.1', 7497, clientId=11)
    conn = sqlite3.connect('options_data.db')

    for ticker in ETF_UNIVERSE:
        contract = Stock(ticker, 'SMART', 'USD')
        ib.qualifyContracts(contract)

        bars = ib.reqHistoricalData(
            contract,
            endDateTime='',
            durationStr='1 Y',        # full year backfill in ONE call
            barSizeSetting='1 day',
            whatToShow='OPTION_IMPLIED_VOLATILITY',
            useRTH=1
        )
        for bar in bars:
            conn.execute(
                "INSERT OR IGNORE INTO iv_history VALUES (?,?,?)",
                (ticker, str(bar.date), bar.close)
            )
        print(f"Backfilled {ticker}: {len(bars)} days")
        time.sleep(1)

    conn.commit(); conn.close(); ib.disconnect()

if __name__ == '__main__':
    run_eod_collection()
BOD Chain Snapshot — Python Script
Runs at 9:31 AM ET. Collects full Greeks/IV/OI for ±10% strikes, 14–120 DTE.
Python — ibkr_bod_snapshot.py 
# ibkr_bod_snapshot.py — Run via cron: 31 9 * * 1-5

from ib_insync import IB, Stock, Option, util
import sqlite3, datetime, time

def run_bod_snapshot():
    ib = IB()
    ib.connect('127.0.0.1', 7497, clientId=20)
    conn = sqlite3.connect('options_data.db')
    today = datetime.date.today()

    for ticker in ETF_UNIVERSE:
        # ── Get current underlying price for ATM strike selection
        stock = Stock(ticker, 'SMART', 'USD')
        ib.qualifyContracts(stock)
        [ticker_data] = ib.reqTickers(stock)
        underlying_px = ticker_data.marketPrice()

        # ── Get option chain expirations
        chains = ib.reqSecDefOptParams(ticker, '', 'STK', stock.conId)
        chain = next(c for c in chains if c.exchange == 'SMART')

        # ── Filter expirations 14–120 DTE
        expiries = [
            e for e in chain.expirations
            if 14 <= (datetime.date(int(e[:4]),int(e[4:6]),int(e[6:])) - today).days <= 120
        ]

        # ── Filter strikes ±10% of spot
        strikes = [
            s for s in chain.strikes
            if underlying_px * 0.90 <= s <= underlying_px * 1.10
        ]

        # ── Snapshot each contract (calls + puts at each strike/expiry)
        for expiry in expiries[:4]:    # limit to 4 nearest expiries
            for strike in strikes:
                for right in ['C', 'P']:
                    opt = Option(ticker, expiry, strike, right, 'SMART')
                    try:
                        ib.qualifyContracts(opt)
                        [tick] = ib.reqMktData(
                            opt,
                            genericTickList='10,11,13',  # bid,ask,model greeks+IV
                            snapshot=True,
                            regulatorySnapshot=False
                        )
                        ib.sleep(0.2)   # allow snapshot to populate
                        conn.execute(
                            """INSERT OR REPLACE INTO chain_snapshots
                               VALUES (?,?,?,?,?,?,?,?,?,?,?,?,?)""",
                            (
                                today, ticker, expiry, strike, right,
                                tick.bid, tick.ask,
                                tick.modelGreeks.impliedVol  if tick.modelGreeks else None,
                                tick.modelGreeks.delta       if tick.modelGreeks else None,
                                tick.modelGreeks.gamma       if tick.modelGreeks else None,
                                tick.modelGreeks.theta       if tick.modelGreeks else None,
                                tick.modelGreeks.vega        if tick.modelGreeks else None,
                                tick.optionOIExch            if tick.optionOIExch else None
                            )
                        )
                    except Exception as e:
                        print(f"Skip {ticker} {expiry} {strike}{right}: {e}")

        time.sleep(0.5)
        print(f"BOD snapshot complete: {ticker}")

    conn.commit(); conn.close(); ib.disconnect()

Bear Case / Risk Assessment
Institutional-grade risk thinking — what could go wrong and how to mitigate.
⚠️
IBKR OPTION_IMPLIED_VOLATILITY methodology is not fully documented. It's IBKR's internal composite — not a transparent vega-weighted ATM average like VIX. For IVR thresholds above 30%, it should be sufficient. But if you're doing precise IVP calibration, small methodology differences vs. ORATS' smoothed IV could cause entry divergence. Mitigate: compare IBKR composite IV vs. manually computed ATM IV (from your BOD snapshot) for 30 days to verify alignment.
⚠️
4:05 PM cron window risk. If IB Gateway crashes or disconnects between 4:00–4:15 PM (e.g., IBKR daily server reset), you miss the day's IV data point. Mitigation: 4:15 PM retry cron job (already in the scheduler above) + health check that alerts you to gaps. A gap of 1–3 days in a 252-day series has minimal IVR impact.
⚠️
TQQQ options chain size. TQQQ (3x leveraged) has a very wide strike range with high OI concentration near spot. The ±10% strike filter should be tighter for TQQQ — recommend ±7% to avoid pulling hundreds of low-liquidity deep OTM strikes. Separate TQQQ config in bod_snapshot.py.
🧠
Behavioral Check — Anchoring Bias: You may anchor to the prior mental model of IBKR as a "pain source" because intraday connectivity was genuinely painful. The BOD/EOD model is architecturally different — stateless, scheduled, 10 minutes total per day. Reframe the mental model before dismissing it. The first 2-week pilot will confirm whether this works cleanly.
