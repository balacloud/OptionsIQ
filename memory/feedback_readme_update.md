name: Update README version at session close
description: README.md version badge must be updated at every session close — part of close checklist
type: feedback
---

Always update the version number and date in README.md at session close (the `> **vX.Y.Z** — Day N` line near the top).

**Why:** README is the first thing visible on GitHub. Leaving it stale makes the repo look abandoned and causes confusion about what version is actually running.

**How to apply:** Add README.md to the session close checklist alongside CLAUDE_CONTEXT.md, KNOWN_ISSUES, ROADMAP, etc. Update the version badge line before committing.
